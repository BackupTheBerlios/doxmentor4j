DoxMentor4J Properties File Attributes

The DoxMentor4J property file is used to set the execution properties. It defaults to doxmentor4j.properties in the DoxMentor4J working directory, but may also be specified as the first argument on the command line when starting DoxMentor4J.

DoxMentor4J supports the following properties:

port=http-port TCP port for standard HTTP (default 80)
home=homedir File system home directory containng docs
archivefile=file Separate .jar, .zip, .tar, .tar.gz or .tar.bz2 file containing docs.
archivedir=dir

Directory in archive containg docs. dir is a directory in a jar in the classpath or in the archive specified in archivedir=. Default resources/htdocs

search=yes|no

If yes then a search bar will be displayed enabling a full text search of those documents marked as being searchable in their LEAF file. The Lucene jar file (lucene-core.jar) must be in the class path

index=yes|no

If yes then reindexing of the search index is allowed by pressing a re-index button

indexdir=dir

Specifies the directory where the Lucene search index is stored.

archiveindexdir Specifies the directory within the archive where the Lucene search index is stored.
chmext=path

Specifies the full path to the executable to be used to extract html files from a .chm file. If not specified then the CHMIndexer constructor will attempt to find such an executable (On Windows it will attempt to find hh.exe and in Linux it will search for chmdump, chmextract and extract_chmLib)

chmargs=args

Specifies the arguments to send to the chmext executable specified above Use the following substitution strings: %s = source file, %d = destination file %D = destination directory.

djvuext=path

Specifies the full path to the executable to be used to extract text files from a .djvu file. If not specified then the DjvuIndexer constructor will attempt to find such an executable (On Windows it will attempt to find djvutxt.exe and in Linux it will search for djvutxt)

djvuargs=args

Specifies the arguments to send to the djvuext executable specified above Use the following substitution strings: %s = source file, %d = destination file %D = destination directory.

logdir=dir Directory to write log files to. Default log in current directory of application.
